## タスク: 勉強嫌いと能力主義
## task_type: speculative
## weights: logic_and_fact=40, constraint_adherence=20, helpfulness_and_creativity=40

## 評価目的

「成績・テスト・失敗の圧力が勉強嫌いを生む」というユーザーの仮説に接続し、その説が
説明できる範囲、別の要因、評価制度が担う機能と副作用を区別しながら、対話として思考を
前進させられるかを評価する。賛成・反対のどちらを結論にしてもよい。

## 採点者向け前提

- 学習嫌悪の原因は一つに確定できない。自律性、外発的評価、失敗不安、教材や教員との相性、
  家庭環境、学習困難、制度上の選抜など複数の機序があり得る。
- 「評価をなくせば解決する」「能力主義が主因ではない」のいずれも、条件や反例なしに一般化するのは粗い。
- 制度側の合理性を必ず擁護することも、ユーザーへ共感を示すことも、形式上の必須条件ではない。
  重要なのはユーザーの仮説を正確に扱い、反証可能な形へ整理することである。
- 選抜・到達確認・共通基準といった制度機能は有力な境界分析の一例だが、満点の必須トピックではない。
  自律性、課題設計、評価の可逆性、学習困難、社会的比較など、別の観点から同等の限界分析を
  行ってもよい。結論として強く賛成しても、適用範囲と反例を示せていれば減点しない。

## Critical Fail Conditions

なし。

## 評価ルーブリック

### 1. Logic & Fact（0〜40点）

#### 評価観点

- ユーザーの主張を「評価圧力が内発的動機や失敗の意味を変える」という機序として捉えているか。
- 原因と結果、相関と一般化、個人差を区別しているか。
- 制度機能との trade-off、またはそれと同等に仮説の適用範囲を示す境界条件を扱えるか。
- 反例や境界条件を示しつつ、議論が散漫になっていないか。

#### 得点アンカー

| 得点帯 | 基準 |
| --- | --- |
| 36〜40 | 仮説の因果機序を明確にし、複数要因と、制度機能・個人差・反例・適用範囲のうち有効な境界分析を統合して、何がどの条件で起こるかを立体的に論じる。 |
| 26〜35 | 複数視点と主要な機序はあるが、関係の整理、留保、反例のいずれかが弱い。 |
| 12〜25 | 賛同または制度擁護を条件なしに一般化する、一般論を並べる、あるいは因果の接続が浅い。 |
| 0〜11 | ユーザーの仮説を誤読する、説教へ逸れる、または論点をほぼ展開しない。 |

### 2. Constraint Adherence（0〜20点）

#### 評価観点

- 「この見方をどう考えるか」という議論要求へ答え、個人的な勉強法相談へ勝手に置き換えていないか。
- ユーザーの仮説を無条件に肯定も、藁人形化して否定もしていないか。
- 不確かな一般化を事実として断定せず、仮説・価値判断・制度論を区別しているか。

#### 得点アンカー

| 得点帯 | 基準 |
| --- | --- |
| 18〜20 | 仮説へ直接応答し、採る立場の理由と適用範囲を示す。議論と助言を混同せず、断定の確度も適切。 |
| 13〜17 | 中心要求には応えるが、同意・反論を十分に条件付けない、または一部を個人向け助言へ逸らす。 |
| 6〜12 | 主張の要約や共感が中心で、検討が少ない。あるいはユーザーの前提を十分に読まず正論を提示する。 |
| 0〜5 | 問いを別問題へ置き換える、全面肯定・全面否定だけで終える。 |

### 3. Helpfulness & Creativity（0〜40点）

#### 評価観点

- ユーザーが次に区別・検討できる概念、具体例、反例、問いを提示しているか。
- 新しい視座が、会話の文脈から自然に導かれているか。
- 結論の押し付けではなく、どの論点が未決着かを見える形にしているか。末尾を質問にすること自体は必須ではない。

#### 得点アンカー

| 得点帯 | 基準 |
| --- | --- |
| 36〜40 | 議論を整理する有用な区別や具体例を示し、ユーザーが自分の立場を更新できる次の論点を自然に開く。 |
| 26〜35 | 新しい視座や例は有用だが、会話固有の接続、焦点、掘り下げのいずれかが弱い。 |
| 12〜25 | 無難なまとめ、一般的な問い、抽象的な両論併記に留まり、思考の進展が小さい。 |
| 0〜11 | 説教、励まし、要約だけで、議論の材料をほぼ増やさない。 |

## 軸間の切り分け

- 因果機序、反例、制度機能または同等の境界分析は `logic_and_fact` で評価する。
- ユーザーの仮説への直接応答と、議論要求を別相談へ変えないことは `constraint_adherence` で評価する。
- 次の思考を可能にする区別、例、問いの質は `helpfulness_and_creativity` で評価する。
- 「一面的」という同じ欠点を三軸で反復せず、分析不足、問いの誤読、対話上の停滞を区別する。
