## タスク: 文体・言語運用の包括評価
## task_type: holistic
## weights: logic_and_fact=40, constraint_adherence=30, helpfulness_and_creativity=30

## 評価目的

複数の非 creative task への回答を横断し、日本語の語義・語用、task に応じた register、文構造、
冗長性、定型表現への依存を評価する。各 task の事実性、論理、提案内容は per-task 評価済みなので
再採点しない。個別の一文ではなく、出力集合に見られる反復傾向と、その読みやすさへの影響を測る。

## 採点者向け前提

- task の性格が違えば、説明調、対話調、コード解説、歴史解説など register が変わるのは自然である。
  「一つの声」は全回答を同じ口調にすることではなく、各文脈での選択が不必要に揺れないことを指す。
- 単発の typo、不自然な一文、一つの foreign term、一つの長い段落を全体傾向へ一般化しない。
- 問題の頻度、複数 task への広がり、意味理解を妨げる程度を分けて評価する。
- 日本語中の英語専門語や code switching は、意味精度に貢献するなら許容する。意図しない文字体系の
  漏出や語義誤用は、意味を乱す場合に減点する。
- 評価対象が少なく横断傾向を判断できない場合、点数を機械的に下げず `confidence` を下げる。
- 空回答、API 失敗、出力上限による明白な途中切れは、可用性・完遂性の欠陥であり、文体傾向ではない。
  その欠落自体をこの rubric で再度減点しない。途中までに実際に現れた文章は語用・文体の証拠に
  できるが、「終わっていないこと」は評価対象から除外する。
- 評価可能な文章が一つもない場合は三軸を 0、Critical Fail は false とする。少数だけある場合は、
  その文章の品質を標本数だけで下げず、横断傾向への confidence を下げる。

## Critical Fail Conditions

なし。文体上の欠点は頻度と影響に応じて三軸で比例的に採点し、全点失効にはしない。

## 評価ルーブリック

### 1. Logic & Fact（0〜40点）

この holistic task に限り、`logic_and_fact` は内容の正誤ではなく、語義・語用・意味制御の正確さを表す。

#### 評価観点

- 文脈に対して意味の合う語、助詞、接続、指示対象を選べているか。
- 抽象語、評価語、因果語を、内容以上に強くまたは曖昧に使っていないか。
- 同じ語の体系的な誤用、翻訳由来の collocation、対象言語に不自然な文字混入が複数出力にあるか。
- 特定の表現を反復しても、意味上必要か、単なる生成癖かを区別できるか。

#### 得点アンカー

| 得点帯 | 基準 |
| --- | --- |
| 36〜40 | 出力集合を通じて語義・指示・接続が精確で、専門語と日常語の選択も文脈に合う。目立つ体系的誤用や意味を薄める定型句依存がない。 |
| 26〜35 | 全体は精確だが、複数箇所に軽い collocation の不自然さ、抽象語の過用、指示の曖昧さがある。理解への影響は小さい。 |
| 12〜25 | 複数 task に語義のずれ、翻訳調、定型句依存が広がり、文意を取り直す場面が繰り返し生じる。 |
| 0〜11 | 体系的な語義誤用や言語漏出が広範で、複数回答の意味理解を継続的に妨げる。 |

### 2. Constraint Adherence（0〜30点）

この holistic task に限り、`constraint_adherence` は各 task の文脈に合う register と、出力間の不要な文体揺れを表す。

#### 評価観点

- technical、historical、speculative、conversational な問いに、適切な硬さ、距離、敬体・常体を選ぶか。
- task 固有の必要がないのに、急に広告調、説教調、過度にくだけた調子、官僚的な調子へ変わらないか。
- 創作を除いた複数回答に、モデル固有の安定した明瞭さがあるか。
- register の変化を「不一致」と機械判定せず、その task に適合するかで判断するか。

#### 得点アンカー

| 得点帯 | 基準 |
| --- | --- |
| 27〜30 | 各 task に合う register を選びつつ、説明姿勢と距離感が安定する。唐突な人格・口調変化がほぼない。 |
| 20〜26 | 大半は適切だが、少数 task で硬すぎる、くだけすぎる、説教的などの局所的な mismatch がある。 |
| 9〜19 | 複数 task で register が目的に合わず、同じモデルの回答として不要な揺れまたは一律な不適合が目立つ。 |
| 0〜8 | ほぼ全体で文脈に合う文体を選べず、回答の受け取り方を大きく損なう。 |

### 3. Helpfulness & Creativity（0〜30点）

この holistic task に限り、`helpfulness_and_creativity` は読みやすさ、文の運び、表現の自然さを表す。

#### 評価観点

- 文と段落の主従関係が明確で、過度な入れ子や nominalization を避けるか。
- 内容に見合う長さで、同じ結論、注意書き、導入句を反復しないか。
- 箇条書き、見出し、文章を必要に応じて使い分け、構造化自体が目的化していないか。
- 定型句を減らすだけでなく、文脈に応じた自然な表現の変化があるか。

#### 得点アンカー

| 得点帯 | 基準 |
| --- | --- |
| 27〜30 | 複数回答が自然な日本語で流れ、段落構造と文の長さが内容に合う。反復や boilerplate が少なく、読む負荷を増やさない。 |
| 20〜26 | 概ね読みやすいが、少数箇所に冗長な前置き、長い入れ子、同じ結び、過剰な箇条書きがある。 |
| 9〜19 | 複数 task で冗長さ、マニュアル調、定型的な見出し、複雑な文構造が反復し、要点把握に追加負荷がかかる。 |
| 0〜8 | 不明瞭な構文と boilerplate が出力集合を支配し、継続的に読解を妨げる。 |

## 軸間の切り分け

- 語の意味、指示、collocation、文字体系の適切さは `logic_and_fact` で評価する。
- task に応じた register、距離感、出力間の不要な揺れは `constraint_adherence` で評価する。
- 文構造、冗長性、段落、反復による読みやすさは `helpfulness_and_creativity` で評価する。
- 同じ定型表現を、語義が薄まる問題、register mismatch、冗長性という異なる影響を確認せず三軸で重複減点しない。
- 一つの outlier の最大影響は原則として局所的な減点に留め、複数 task に広がる pattern と区別する。
- 空回答や途中切れを per-task の完遂性欠陥として既に扱っている場合、それ自体を文体・register・
  読みやすさの欠点へ読み替えて重複減点しない。
