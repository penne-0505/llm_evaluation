## タスク: 人間が認識するもの全てへのID付与
## task_type: speculative
## weights: logic_and_fact=40, constraint_adherence=20, helpfulness_and_creativity=40

## 評価目的

物理物だけでなく、出来事、概念、知覚、関係、同一対象の別の見え方まで含む極端な思考実験として
問いを受け止め、「ID は何に付くのか」「同一性を誰がどの文脈で決めるのか」を明示した上で、
大胆な理論または実装可能なアーキテクチャを一貫して展開できるかを評価する。

## 採点者向け前提

- 「世界そのものの絶対 ID」と、「人間または system が作った表象・観測・claim の ID」は異なる。
- 同一対象でも、時刻、観測者、粒度、文脈、version によって別の認識が生じる。type / token、
  entity / event、class / instance、object / representation を区別する設計が必要になり得る。
- ID を付けるには、identity criterion、namespace、scope、versioning、equivalence relation、provenance、
  access control などの選択が不可避である。
- URI、RDF、knowledge graph、content hash、event sourcing、sensor ID などは利用可能な例であり必須語ではない。
- クオリアや曖昧な概念を「直接」一意に同定できるかは未解決であり、観測記録や報告へ ID を付ける
  という回避策も妥当である。

## Critical Fail Conditions

なし。

## 評価ルーブリック

### 1. Logic & Fact（0〜40点）

#### 評価観点

- ID の対象を、物、状態、出来事、概念、知覚、表象のどこに置くか定義しているか。
- 同一性、粒度、時間、観測者、文脈の問題を、選んだ方式の中で一貫して扱うか。
- 絶対的同一性を仮定する場合、その仮定と帰結を自覚しているか。表象へ限定する場合、問いを矮小化せず理由を示すか。
- 物理的・概念的どちらの方向でも、スケールと衝突する点を論理的に掘り下げているか。

#### 得点アンカー

| 得点帯 | 基準 |
| --- | --- |
| 36〜40 | ID の担体と identity criterion を明示し、時間・文脈・観測者・粒度の少なくとも複数を統合した一貫したモデルを展開する。大胆案でも実務案でもよい。 |
| 26〜35 | 中心モデルと限界は明確だが、同一性、version、表象との区別のいずれかが未整理。 |
| 12〜25 | 技術や哲学概念を複数挙げるが、何に ID が付くかが曖昧、または物理物だけへ狭める。 |
| 0〜11 | バーコードや個人番号の話だけで問いのスケールを扱わない、または一貫したアプローチがない。 |

### 2. Constraint Adherence（0〜20点）

#### 評価観点

- 「人間が認識する全て」という極端な範囲を正面から扱い、難しいという理由だけで通常の資産管理へ縮小していないか。
- 実現不能な部分、倫理・privacy、権力、誤同定、namespace 衝突などの限界を、提案と対応付けているか。
- 思考実験としての自由度を保ちつつ、前提を暗黙にすり替えていないか。

#### 得点アンカー

| 得点帯 | 基準 |
| --- | --- |
| 18〜20 | 極端な前提を維持したまま、可能な層と不可能・未定義な層を明示し、提案の限界と社会的帰結を具体的に扱う。 |
| 13〜17 | 範囲と限界は扱うが、特定領域への縮小、倫理面、認識の主観性のいずれかが弱い。 |
| 6〜12 | 「膨大で難しい」と述べるだけ、または物理 object へのタグ付けへ実質的に置き換える。 |
| 0〜5 | 問いの前提を無視し、別スケールの ID 制度だけを説明する。 |

### 3. Helpfulness & Creativity（0〜40点）

#### 評価観点

- 読者が model を試せる具体例、thought experiment、data flow、architecture を示すか。
- 既存技術を紹介するだけでなく、それをどこまで拡張し、どこで破綻するかを見せるか。
- 「絶対 ID」「観測イベント ID」「claim ID」など複数案がある場合、選択基準や trade-off を示すか。
- 大胆さと明瞭さのどちらかを犠牲にせず、問いの面白さを増しているか。

#### 得点アンカー

| 得点帯 | 基準 |
| --- | --- |
| 36〜40 | 一貫した architecture または thought experiment を具体例で展開し、既存技術との接続、破綻点、社会的含意まで新しい見方を与える。 |
| 26〜35 | 面白い中心案と具体例があるが、trade-off、実装像、哲学的帰結のいずれかが弱い。 |
| 12〜25 | 既存概念の紹介または大胆なアイデアはあるが、独自展開と具体性の片方が不足する。 |
| 0〜11 | 「不可能」「難しい」で終わる、または既存技術名を並べるだけで思考を展開しない。 |

## 軸間の切り分け

- ID の担体、同一性、時間・文脈を含むモデル整合性は `logic_and_fact` で評価する。
- 問いの極端な範囲を維持し、限界と前提を明示することは `constraint_adherence` で評価する。
- architecture、具体例、thought experiment、trade-off の発見は `helpfulness_and_creativity` で評価する。
- 物理物へ矮小化した欠点は、モデル上の欠落、問いのすり替え、展開の乏しさという別の影響がある場合だけ複数軸へ反映する。
