## タスク: おいしさの圏論的表現
## task_type: creative
## weights: logic_and_fact=30, constraint_adherence=30, helpfulness_and_creativity=40

## 評価目的

「料理のおいしさ」を、圏論の用語を飾りとして並べるのではなく、対象・射・合成・関手などの
関係構造を保った比喩として説明し、数学の専門外でも直感を追える形にできるかを評価する。
一つの正規マッピングは存在せず、別の圏論的構成も整合していれば高く評価する。

## 採点者向け前提

- 圏では、対象と射があり、射には恒等射と結合的な合成がある。
- 関手を用いるなら、対象と射を別の圏へ写し、恒等射と合成を保つという役割がある。
- 料理・知覚との対応は通常は比喩またはモデルであり、数学的定理として証明されたものではない。
- 「食材＝対象、調理＝射、味覚＝関手」は一例にすぎない。状態遷移、レシピ、食べ手、文脈、
  評価の変化などを別の構造で表してもよい。
- 「おいしさ」は、食べ手への写像、料理間の選好順序、状態や射への評価、豊穣圏の値など、
  複数の置き場所を選べる。観測者依存性を扱うことは有力だが必須ではなく、採用したモデルの
  前提と限界を明示し、一貫して説明できることを重視する。

## Critical Fail Conditions

なし。

## 評価ルーブリック

### 1. Logic & Fact（0〜30点）

#### 評価観点

- 選んだ対象・射・合成の対応が途中で変わらず、恒等や合成の直感と矛盾しないか。
- 関手、自然変換、積など追加概念を使う場合、その役割を大きく取り違えていないか。
- 「おいしさ」の生成、比較、評価または観測が、選んだ構造のどこに現れるかを説明しているか。
- 比喩と厳密な数学的主張の境界を必要な範囲で示しているか。

#### 得点アンカー

| 得点帯 | 基準 |
| --- | --- |
| 27〜30 | 少なくとも中心となる圏論構造が一貫し、料理の過程と、おいしさの生成・比較・評価・観測のいずれかが同じモデル内で接続される。追加概念を使っても役割が整合する。 |
| 20〜26 | 中心マッピングは追えるが、合成、関手、評価の置き場所など一部の対応が曖昧または途中で揺れる。 |
| 9〜19 | 圏論用語と料理の対応をいくつか示すが、用語間の関係が弱く、構造より単語の対応表に近い。 |
| 0〜8 | 用語の意味が大きく崩れる、対応が互いに矛盾する、または数学用語の羅列に留まる。 |

### 2. Constraint Adherence（0〜30点）

#### 評価観点

- 主題が「料理一般」ではなく「おいしさ」の説明になっているか。
- 「わかりやすく」という要求に応え、専門用語を使う際に短い直感的説明を添えているか。
- 選んだモデルが何を表し、何を捨象するかを読者が把握できるか。食べ手や文脈を採用する場合は、物質変化との関係を説明しているか。
- 厳密さを装うために不要な形式記号を増やしていないか。

#### 得点アンカー

| 得点帯 | 基準 |
| --- | --- |
| 27〜30 | 「おいしさ」を選んだ構造内へ明示的に置き、そのモデルの前提と限界を、専門外にも追える説明で扱う。食べ手依存を採るかどうか自体は問わない。 |
| 20〜26 | 分かりやすく主題にも答えるが、おいしさの位置、モデルの前提、用語説明のいずれかが一部弱い。 |
| 9〜19 | 料理工程の圏論化が中心で、おいしさが添え物になる。あるいは専門用語への依存が大きい。 |
| 0〜8 | 問いを別の数学説明へ置き換える、または非専門家が対応関係を追えない。 |

### 3. Helpfulness & Creativity（0〜40点）

#### 評価観点

- 一つの料理や一口の体験を通じて、抽象概念を具体的に見せているか。
- 比喩が「関係と合成を見る」という圏論らしい視点を読者へ与えるか。
- 正確さを保ちながら、説明として面白く、記憶に残る発見があるか。

#### 得点アンカー

| 得点帯 | 基準 |
| --- | --- |
| 36〜40 | 具体例と構造が相互に支え合い、圏論を知らない読者にも新しい見方として腑に落ちる。独自性が説明を明瞭にしている。 |
| 26〜35 | 有用な例と比喩があり概念を追えるが、洞察の深さ、具体性、表現の鮮度のいずれかが弱い。 |
| 12〜25 | 対応表としては理解できるが、例が薄い、説明が教科書的、または比喩が概念理解を十分に助けない。 |
| 0〜11 | 読者への橋渡しがなく、難解さまたはワードサラダが支配的。 |

## 軸間の切り分け

- 圏論概念とマッピングの整合性は `logic_and_fact` で評価する。
- 「おいしさ」「わかりやすさ」、選んだモデルの前提と限界への応答は `constraint_adherence` で評価する。
- 具体例、直感、比喩としての発見と説明力は `helpfulness_and_creativity` で評価する。
- 用語誤用を Logic と Helpfulness の双方で同じ理由のまま重複減点せず、数学的破綻と理解困難という別の影響がある場合だけ分ける。
