# ダッシュボード拡張計画: 評価履歴と詳細比較

**タスクID**: UI-Feature-15-Ext  
**親タスク**: UI-Feature-15  
**作成日**: 2026-02-26  

---

## 1. 目標 (Goal)

ダッシュボードに「評価履歴」と「詳細比較」機能を追加し、サイドバーの「過去の結果」を統合する。

---

## 2. スコープ (Scope)

### In Scope
- 📋 評価履歴タブ（新規）
  - カード形式での評価結果一覧表示
  - ページネーション（4件/ページ）
  - 前へ/次へナビゲーション
  - カードクリックで詳細表示（タスク別結果/横断サマリー）
- 🔍 詳細比較タブ（新規）
  - 2モデル選択による比較
  - 左右並列表示（タスク別）
  - 折りたたみ表示（スコアは常時表示）
  - 「未実行」明示
- サイドバー「過去の結果」削除
- データ読み込み：都度読み込み（キャッシュなし）

### Out of Scope
- 3モデル以上の同時比較
- 比較結果のエクスポート
- フィルタ・検索機能

## 3. 設計 (Design)

### 3.1 アーキテクチャ

```
ui/components.py
├── render_dashboard()              # メインエントリ
│   ├── 📈 スコア比較（既存）
│   ├── 📋 評価履歴（新規）
│   │   ├── _render_history_cards()    # カード一覧
│   │   ├── _render_history_detail()   # 詳細表示
│   │   └── _load_all_results()        # 全結果読み込み
│   └── 🔍 詳細比較（新規）
│       ├── _render_comparison_select() # 選択UI
│       └── _render_comparison_view()   # 比較表示
└── [既存関数は維持]

app.py
└── render_sidebar()
    └── [過去の結果セクションを削除]
```

### 3.2 データフロー

```
【評価履歴】
results/*.json → _load_all_results() → DataFrame → ページネーション → カード表示
                                                  ↓
                                           カードクリック
                                                  ↓
                                        ResultStorage.load() → 詳細表示

【詳細比較】
Model A選択 ─┐
             ├→ ファイル読み込み → タスク一覧マージ → 左右表示
Model B選択 ─┘
```

## 5. 技術仕様

### 5.1 ファイル読み込み

| 項目 | 仕様 |
|------|------|
| パス | `results/*.json` |
| ソート | `executed_at`降順 |
| 最大件数 | 全件（パフォーマンス次第で制限検討） |
| キャッシュ | なし（都度読み込み） |

### 5.2 ページネーション

| 項目 | 仕様 |
|------|------|
| 1ページ件数 | 4件 |
| ナビゲーション | 前へ / 次へ ボタン |
| ページ表示 | 「ページ X/Y」形式 |

### 5.3 比較機能

| 項目 | 仕様 |
|------|------|
| 比較可能数 | 2モデル固定 |
| タスクマージ | タスクIDで結合 |
| 未実行表示 | 「【未実行】このタスクは実行されていません」 |
| デフォルト展開 | expanderは折りたたみ状態 |
