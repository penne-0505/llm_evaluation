# ダッシュボード表示実装計画

**タスクID**: UI-Feature-15  
**関連タスク**: なし  
**作成日**: 2026-02-26  

---

## 1. 目標 (Goal)

待機中状態のメインエリアに、過去の評価結果を表示するダッシュボードを実装し、デッドスペースを有効活用する。

---

## 2. スコープ (Scope)

### In Scope
- `results/` ディレクトリ内のJSONファイル読み込みと解析
- 最新5件の評価結果カード表示
- モデル別平均スコア比較（棒グラフ）
- モデル別比較テーブル
- クイックアクションボタン

### Out of Scope
- リアルタイム更新（手動更新のみ）
- データの永続化（既存のJSONファイルを使用）
- ROI可視化グラフ（将来の拡張案としてTODO.mdに記載済み）

---

## 3. 設計 (Design)

### 3.1 アーキテクチャ

```
┌─────────────────────────────────────┐
│  app.py メインエリア                │
│  ┌───────────────────────────────┐  │
│  │  render_dashboard()           │  │
│  │  ├─ _load_recent_results()   │  │
│  │  ├─ _render_result_cards()   │  │
│  │  ├─ _render_score_bar_chart()│  │
│  │  └─ _render_model_table()    │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
           │
           ▼
┌─────────────────────────────────────┐
│  results/*.json                     │
└─────────────────────────────────────┘
```

### 3.2 データフロー

1. `results/`ディレクトリをスキャン
2. JSONファイルをパースしてサマリー情報を抽出
3. データを集計・整形
4. キャッシュに保存（TTL: 60秒）
5. 各コンポーネントにデータを渡して表示

### 3.3 データ構造

```python
# 単一結果ファイルのサマリー
ResultSummary = {
    "filename": str,
    "target_model": str,
    "executed_at": datetime,
    "task_count": int,
    "judge_count": int,
    "avg_score": float,  # 全タスクのtotal_score_meanの平均
    "max_score": float,
    "min_score": float,
}

# モデル別集計
ModelStats = {
    "model_name": str,
    "run_count": int,
    "avg_score": float,
    "best_score": float,
    "latest_run": datetime,
}
```

## 5. 技術仕様

### 5.1 ファイル読み込み仕様

| 項目 | 仕様 |
|------|------|
| パス | `results/*.json` |
| ソート | `executed_at`降順 |
| 最大件数 | 50件 |
| キャッシュTTL | 60秒 |
| エラー処理 | JSON破損ファイルはスキップ、ログ出力 |

### 5.2 依存関係

- **既存**: `plotly`, `pandas`, `streamlit`
- **新規**: なし（標準ライブラリのみ）

### 5.3 パフォーマンス考慮事項

- JSONファイルの大きい場合は必要フィールドのみ読み込み
- グラフ描画は最大20モデルまで（それ以上は上位20モデル）
- 画像ファイルの遅延読み込み

---

## 6. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| 結果ファイルが多い場合の読み込み遅延 | 中 | キャッシュ活用、最大50件制限 |
| JSONスキーマ変更 | 低 | 必要フィールドのみ抽出、柔軟なパース |
| 同時アクセスによるファイルロック | 低 | 読み取り専用、例外キャッチ |
