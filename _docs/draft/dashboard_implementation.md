# メインエリアダッシュボード実装案

## 概要

待機中状態のメインエリアに、過去の評価結果を表示するダッシュボードを実装する。

## 目的

- デッドスペースの有効活用
- 過去の評価結果への簡易アクセス提供
- 評価傾向の可視化

## 機能要件

### 1. 過去結果サマリー表示

- `results/` ディレクトリ内のJSONファイルを読み込み
- 最新5件をカード形式で表示
- 各カードに以下を表示：
  - 評価対象モデル名
  - 実行日時（相対表示："3日前"など）
  - 平均スコア（全タスクのtotal_score_meanの平均）
  - 評価タスク数
  - judgeモデル数

### 2. モデル別平均スコア比較（棒グラフ）

- 評価済みモデルの平均スコアを棒グラフで表示
- X軸：モデル名（短縮表示）
- Y軸：平均スコア（0-100）
- 同一モデルの複数評価がある場合は平均値を使用
- スコア帯で色分け（優秀:緑、良好:黄、要改善:赤）
- Plotlyを使用（既存依存関係に含まれる）
- ホバー時：評価回数、最高/最低スコアを表示

### 3. モデル別比較

- 評価済みモデルの一覧をテーブル表示
- 列：モデル名、評価回数、最高スコア、平均スコア、最新評価日
- ソート可能

### 4. クイックアクション

- 「結果を詳細表示」ボタン（既存の結果読み込み機能へリンク）
- 「新規評価を開始」ボタン（サイドバーにフォーカス）

## 実装詳細

### コンポーネント構成

```
ui/components.py
├── render_dashboard()              # ダッシュボード全体
├── _load_recent_results()          # 結果ファイル読み込み（内部関数）
├── _render_result_cards()          # 結果カード表示
├── _render_model_score_bar_chart() # モデル別スコア棒グラフ
└── _render_model_comparison()      # モデル別比較テーブル
```

### データ構造

```python
# 単一結果ファイルのサマリー
ResultSummary = {
    "filename": str,
    "target_model": str,
    "executed_at": datetime,
    "task_count": int,
    "judge_count": int,
    "avg_score": float,  # 全タスクのtotal_score_meanの平均
    "max_score": float,
    "min_score": float,
}

# モデル別集計
ModelStats = {
    "model_name": str,
    "run_count": int,
    "avg_score": float,
    "best_score": float,
    "latest_run": datetime,
}
```

### ファイル読み込み仕様

- パス：`results/*.json`
- ソート：`executed_at`の降順
- 最大件数：最新50件（パフォーマンス考慮）
- エラーハンドリング：JSON破損ファイルはスキップしてログ出力

### 表示条件

- 結果ファイルが存在する場合：ダッシュボードを表示
- 結果ファイルが存在しない場合：
  - 「まだ評価結果がありません」メッセージ
  - クイックスタートガイド（簡易版）

### パフォーマンス考慮事項

- ファイル読み込みは`@st.cache_data`でキャッシュ（TTL: 60秒）
- グラフ描画は件数が多い場合はサンプリング（最新20件）
- 大きなJSONファイルは必要なフィールドのみ読み込み

## 実装ステップ

1. `ui/components.py`に`render_dashboard()`関数を追加
2. `app.py`のメインエリアにダッシュボード表示ロジックを追加
   - 評価実行中はダッシュボードを非表示
   - 待機中のみダッシュボードを表示
3. 結果ファイル読み込みユーティリティを実装
4. グラフ描画コンポーネントを実装
5. テスト（複数結果ファイルがある状態で表示確認）

## 依存関係

- 既存：`plotly`, `pandas`, `streamlit`
- 新規：なし（標準ライブラリのみ）
